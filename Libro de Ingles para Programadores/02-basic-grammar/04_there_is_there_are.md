# There Is / There Are (Hay)

## Introducción

En español usamos una sola palabra para expresar existencia: **"hay"**. En inglés debemos elegir entre **"there is"** (singular) y **"there are"** (plural). Esta estructura es extremadamente común en programación para describir qué existe o no existe en un sistema.

---

## Regla fundamental

| Estructura | Uso | Ejemplo |
|------------|-----|---------|
| **There is** | Sustantivo singular o incontable | There is a bug in the code |
| **There are** | Sustantivo plural | There are three errors in the log |

### Contracciones

| Forma completa | Contracción | Ejemplo |
|----------------|-------------|---------|
| There is | There's | There's a new commit |
| There are | (no se contrae normalmente) | There are two branches |

---

## Forma afirmativa en programación

### There is (singular)

| Inglés | Español |
|--------|---------|
| There is a bug in the code | Hay un bug en el código |
| There is a new version available | Hay una nueva versión disponible |
| There is a README in the repository | Hay un README en el repositorio |
| There is an error in line 42 | Hay un error en la línea 42 |
| There is a conflict in the merge | Hay un conflicto en el merge |
| There is a missing semicolon | Hay un punto y coma faltante |
| There is a security vulnerability | Hay una vulnerabilidad de seguridad |
| There is a pull request waiting for review | Hay un pull request esperando revisión |

### There are (plural)

| Inglés | Español |
|--------|---------|
| There are no errors in the console | No hay errores en la consola |
| There are five open issues | Hay cinco issues abiertos |
| There are two possible solutions | Hay dos soluciones posibles |
| There are several deprecated methods | Hay varios métodos deprecados |
| There are many frameworks to choose from | Hay muchos frameworks para elegir |
| There are three failing tests | Hay tres tests fallando |
| There are no breaking changes | No hay cambios que rompan compatibilidad |
| There are some unused variables | Hay algunas variables sin usar |

## Forma negativa

### Con "no"

```
There is no + sustantivo singular
There are no + sustantivo plural
```

| Inglés | Español |
|--------|---------|
| There is no documentation | No hay documentación |
| There is no error message | No hay mensaje de error |
| There are no tests for this module | No hay tests para este módulo |
| There are no open pull requests | No hay pull requests abiertos |

### Con "not" (+ a/any)

```
There is not (isn't) + a + sustantivo singular
There are not (aren't) + any + sustantivo plural
```

| Inglés | Español |
|--------|---------|
| There isn't a config file | No hay un archivo de configuración |
| There isn't a default value | No hay un valor por defecto |
| There aren't any comments in the code | No hay ningún comentario en el código |
| There aren't any dependencies | No hay ninguna dependencia |

> **Nota:** "There is no" y "There isn't a" significan lo mismo. "There is no" es ligeramente más enfático.

---

## Forma interrogativa

```
Is there + sustantivo singular?
Are there + sustantivo plural?
```

| Pregunta | Traducción |
|----------|------------|
| Is there a README? | ¿Hay un README? |
| Is there a way to fix this? | ¿Hay una forma de arreglar esto? |
| Is there an alternative library? | ¿Hay una librería alternativa? |
| Is there a test for this function? | ¿Hay un test para esta función? |
| Are there any known issues? | ¿Hay algún issue conocido? |
| Are there any breaking changes? | ¿Hay algún cambio que rompa compatibilidad? |
| Are there any open tickets? | ¿Hay algún ticket abierto? |
| Are there any unused imports? | ¿Hay algún import sin usar? |

### Respuestas cortas

| Pregunta | Sí | No |
|----------|----|----|
| Is there a bug? | Yes, there is | No, there isn't |
| Are there any errors? | Yes, there are | No, there aren't |

---

## Uso con "some", "any", "many", "several"

| Palabra | Uso | Ejemplo |
|---------|-----|---------|
| some | Afirmativo (algunos) | There are some warnings |
| any | Preguntas y negaciones | Are there any updates? / There aren't any issues |
| many | Pregunta/negación (muchos) | Are there many contributors? |
| several | Afirmativo (varios) | There are several options |
| a few | Afirmativo (unos pocos) | There are a few edge cases |
| a lot of | Afirmativo (muchos) | There are a lot of files to review |

## Contextos comunes en programación

### En mensajes de error y logs

```
Error: There is no such file or directory.
Warning: There are unused variables in this scope.
Info: There is a new update available.
Error: There is no default export in this module.
```

### En code reviews

```
There is a typo on line 15.
There are some magic numbers that should be constants.
There is no error handling in this function.
There are too many nested conditions.
```

### En reuniones y standups

```
There is a blocker on the authentication feature.
There are two PRs waiting for review.
There is a deployment scheduled for tonight.
There are some performance issues we need to address.
```

---

## Errores comunes de hispanohablantes

| Error | Correcto | Explicación |
|-------|----------|-------------|
| ~~Have a bug in the code~~ | There is a bug in the code | "Hay" no se traduce como "have" |
| ~~There is three errors~~ | There are three errors | Plural requiere "are" |
| ~~Exist a problem~~ | There is a problem | "Hay" no se traduce como "exist" |
| ~~Are there a README?~~ | Is there a README? | Singular requiere "Is" |
| ~~There are a bug~~ | There is a bug | Singular requiere "is" |

## Ejercicios

### Ejercicio 1: Completa con "there is" o "there are"

1. ___ a syntax error in the code.
2. ___ five open issues on GitHub.
3. ___ no documentation for this API.
4. ___ several ways to solve this.
5. ___ a meeting at 10 AM.
6. ___ no unit tests in this project.

### Ejercicio 2: Forma preguntas

1. There is a config file. → ___?
2. There are pending migrations. → ___?
3. There is an API key in the .env file. → ___?
4. There are unused dependencies. → ___?

### Ejercicio 3: Convierte a negativo

1. There is a default value. → ___
2. There are comments in the code. → ___
3. There is a test for this function. → ___
4. There are breaking changes. → ___

### Ejercicio 4: Traduce al inglés

1. Hay un bug en la línea 30.
2. No hay errores en la consola.
3. ¿Hay un README en el repositorio?
4. Hay varias soluciones posibles.
5. No hay tests para este módulo.

---

**Ejercicio 1:** 1. There is, 2. There are, 3. There is, 4. There are, 5. There is, 6. There are

3. Is there an API key in the .env file?
4. Are there unused dependencies?

**Ejercicio 3:**

2. There are no comments in the code. / There aren't any comments in the code.
