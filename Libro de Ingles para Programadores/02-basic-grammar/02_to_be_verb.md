# El Verbo "To Be" (Ser / Estar)

## Introducción

El verbo **"to be"** es el más importante y usado del inglés. Significa tanto **"ser"** como **"estar"**. En programación lo usamos constantemente para describir estados, condiciones y definiciones.

---

## Conjugación en presente

| Sujeto | Verbo | Contracción | Ejemplo |
|--------|-------|-------------|---------|
| I | am | I'm | I am a developer / I'm a developer |
| You | are | You're | You are online / You're online |
| He | is | He's | He is the tech lead / He's the tech lead |
| She | is | She's | She is a senior dev / She's a senior dev |
| It | is | It's | It is null / It's null |
| We | are | We're | We are a team / We're a team |
| They | are | They're | They are passing / They're passing |

---

## Usos en programación

### 1. Describir el estado de algo

| Inglés | Español |
|--------|---------|
| The variable **is** null | La variable **es/está** nula |
| The server **is** down | El servidor **está** caído |
| The tests **are** passing | Los tests **están** pasando |
| The API **is** available | La API **está** disponible |
| The connection **is** secure | La conexión **es** segura |
| The file **is** empty | El archivo **está** vacío |
| The build **is** broken | El build **está** roto |
| The response **is** valid | La respuesta **es** válida |

### 2. Definir identidad o rol

| Inglés | Español |
|--------|---------|
| I **am** a frontend developer | Yo **soy** un desarrollador frontend |
| She **is** the project manager | Ella **es** la project manager |
| We **are** the QA team | Nosotros **somos** el equipo de QA |
| This **is** a REST API | Esto **es** una API REST |
| Python **is** a programming language | Python **es** un lenguaje de programación |

### 3. Describir características

| Inglés | Español |
|--------|---------|
| The code **is** clean | El código **está** limpio |
| The function **is** pure | La función **es** pura |
| The algorithm **is** efficient | El algoritmo **es** eficiente |
| The documentation **is** outdated | La documentación **está** desactualizada |
| The framework **is** lightweight | El framework **es** ligero |

## Forma negativa

Se forma añadiendo **"not"** después del verbo "to be".

| Forma completa | Contracción | Ejemplo |
|----------------|-------------|---------|
| I am not | I'm not | I'm not available right now |
| You are not | You aren't / You're not | You aren't assigned to this task |
| He is not | He isn't / He's not | He isn't the repo owner |
| She is not | She isn't / She's not | She isn't in the meeting |
| It is not | It isn't / It's not | It isn't working / It's not working |
| We are not | We aren't / We're not | We aren't ready to deploy |
| They are not | They aren't / They're not | They aren't passing |

### Ejemplos negativos en programación

```
The variable is not defined.        → La variable no está definida.
The server is not responding.       → El servidor no está respondiendo.
The tests are not passing.          → Los tests no están pasando.
The endpoint is not available.      → El endpoint no está disponible.
The feature is not implemented yet. → La funcionalidad no está implementada aún.
The bug is not reproducible.        → El bug no es reproducible.
```

## Forma interrogativa

Se forma invirtiendo el sujeto y el verbo: **"Is/Are + Subject...?"**

| Afirmativa | Interrogativa |
|------------|---------------|
| The server is running | **Is** the server running? |
| The tests are passing | **Are** the tests passing? |
| It is a bug | **Is** it a bug? |
| You are available | **Are** you available? |
| They are ready | **Are** they ready to merge? |

### Preguntas comunes en programación

| Pregunta | Traducción |
|----------|------------|
| Is the server running? | ¿Está corriendo el servidor? |
| Is this a known bug? | ¿Es este un bug conocido? |
| Are the tests passing? | ¿Están pasando los tests? |
| Is the database connected? | ¿Está conectada la base de datos? |
| Are you available for a code review? | ¿Estás disponible para un code review? |
| Is the PR ready for review? | ¿Está el PR listo para revisión? |
| Is the deployment successful? | ¿Fue exitoso el despliegue? |

---

## Respuestas cortas con "to be"

En inglés, las respuestas cortas son muy comunes y se consideran más naturales.

| Pregunta | Respuesta afirmativa | Respuesta negativa |
|----------|---------------------|--------------------|
| Is it working? | Yes, it is | No, it isn't |
| Are you free? | Yes, I am | No, I'm not |
| Is she online? | Yes, she is | No, she isn't |
| Are they ready? | Yes, they are | No, they aren't |

> **Nota:** Nunca se contrae la respuesta afirmativa corta. "Yes, it's" es INCORRECTO. Se dice "Yes, it is."

---

## Errores comunes de hispanohablantes

| Error | Correcto | Explicación |
|-------|----------|-------------|
| ~~I am agree~~ | I agree | "Agree" es verbo, no adjetivo |
| ~~It is have a bug~~ | It has a bug | No mezclar "to be" con "to have" |
| ~~The code is work~~ | The code works / The code is working | "Work" es verbo, no adjetivo |
| ~~Is a good framework~~ | It is a good framework | Falta el sujeto "It" |
| ~~Are 5 errors~~ | There are 5 errors | Se necesita "There" |
