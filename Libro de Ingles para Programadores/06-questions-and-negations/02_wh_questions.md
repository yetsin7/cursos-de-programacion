# WH Questions

## Preguntas con palabras interrogativas en contexto técnico

Las WH Questions son preguntas abiertas que comienzan con una palabra interrogativa. Son esenciales para obtener información específica durante reuniones, debugging, code reviews y conversaciones técnicas.

---

## 1. Las 7 palabras interrogativas principales

| Palabra | Significado | Uso principal |
|---------|------------|---------------|
| **What** | ¿Qué? / ¿Cuál? | Pedir información o definición |
| **Where** | ¿Dónde? | Ubicación (archivos, servidores, código) |
| **When** | ¿Cuándo? | Tiempo, fechas, plazos |
| **Why** | ¿Por qué? | Razones, causas de errores |
| **How** | ¿Cómo? | Método, proceso, instrucciones |
| **Who** | ¿Quién? | Personas, responsables |
| **Which** | ¿Cuál? (entre opciones) | Elegir entre alternativas |

---

## 2. Estructura general

```
WH word + auxiliar + sujeto + verbo + complemento?
```

Ejemplos de estructura:
```
What   + does + this function + do?
Where  + is   + the config file?
Why    + is   + the build      + failing?
How    + do   + I              + install dependencies?
```

---

## 3. WHAT — ¿Qué? / ¿Cuál?

Usamos **What** para preguntar por información, definiciones o detalles.

| Pregunta | Traducción |
|----------|------------|
| What does this function do? | ¿Qué hace esta función? |
| What is the return type? | ¿Cuál es el tipo de retorno? |
| What version are you using? | ¿Qué versión estás usando? |
| What error are you getting? | ¿Qué error te aparece? |
| What language is this written in? | ¿En qué lenguaje está escrito esto? |
| What happens if the request times out? | ¿Qué pasa si la solicitud expira? |
| What do you think about this approach? | ¿Qué opinas de este enfoque? |
| What's the expected behavior? | ¿Cuál es el comportamiento esperado? |

---

## 4. WHERE — ¿Dónde?

Usamos **Where** para preguntar por ubicaciones: archivos, carpetas, servidores, código.

| Pregunta | Traducción |
|----------|------------|
| Where is the config file? | ¿Dónde está el archivo de configuración? |
| Where do I find the API keys? | ¿Dónde encuentro las API keys? |
| Where is this function defined? | ¿Dónde está definida esta función? |
| Where should I add the new route? | ¿Dónde debería agregar la nueva ruta? |
| Where are the logs stored? | ¿Dónde se almacenan los logs? |
| Where does this component get its data? | ¿De dónde obtiene sus datos este componente? |

---

## 5. WHEN — ¿Cuándo?

Usamos **When** para preguntar por tiempos, plazos y momentos.

| Pregunta | Traducción |
|----------|------------|
| When is the deadline? | ¿Cuándo es la fecha límite? |
| When was the last deploy? | ¿Cuándo fue el último deploy? |
| When do we need to ship this? | ¿Cuándo necesitamos entregar esto? |
| When did the server crash? | ¿Cuándo se cayó el servidor? |
| When will the fix be ready? | ¿Cuándo estará listo el fix? |
| When does the sprint end? | ¿Cuándo termina el sprint? |

---

## 6. WHY — ¿Por qué?

Usamos **Why** para preguntar razones. Es fundamental durante debugging y code reviews.

| Pregunta | Traducción |
|----------|------------|
| Why is the build failing? | ¿Por qué está fallando el build? |
| Why are we using this library? | ¿Por qué estamos usando esta librería? |
| Why does this test fail? | ¿Por qué falla este test? |
| Why was this deprecated? | ¿Por qué fue deprecado esto? |
| Why don't we use TypeScript? | ¿Por qué no usamos TypeScript? |
| Why is the response so slow? | ¿Por qué es tan lenta la respuesta? |
| Why did you choose this approach? | ¿Por qué elegiste este enfoque? |

---

## 7. HOW — ¿Cómo?

Usamos **How** para preguntar por métodos, procesos e instrucciones. También se combina con otras palabras.

### How solo

| Pregunta | Traducción |
|----------|------------|
| How do I install dependencies? | ¿Cómo instalo las dependencias? |
| How does this algorithm work? | ¿Cómo funciona este algoritmo? |
| How do I run the tests? | ¿Cómo ejecuto los tests? |
| How did you fix the bug? | ¿Cómo arreglaste el bug? |

### How + adjetivo / adverbio

| Combinación | Significado | Ejemplo |
|------------|-------------|---------|
| How many | ¿Cuántos? | How many endpoints does this API have? |
| How much | ¿Cuánto? | How much memory does it use? |
| How long | ¿Cuánto tiempo? | How long does the build take? |
| How often | ¿Con qué frecuencia? | How often do you deploy? |
| How far | ¿Qué tan lejos/avanzado? | How far along is the migration? |

---

## 8. WHO — ¿Quién?

Usamos **Who** para preguntar por personas y responsables.

| Pregunta | Traducción |
|----------|------------|
| Who wrote this code? | ¿Quién escribió este código? |
| Who is responsible for the backend? | ¿Quién es responsable del backend? |
| Who should I ask about the database? | ¿A quién debería preguntarle sobre la base de datos? |
| Who reviewed this PR? | ¿Quién revisó este PR? |
| Who is on call this week? | ¿Quién está de guardia esta semana? |

> **Nota:** Cuando **Who** es el sujeto, no se usa auxiliar:
> **Who wrote this?** (no: *Who did write this?*)

---

## 9. WHICH — ¿Cuál? (entre opciones)

Usamos **Which** cuando hay opciones limitadas para elegir.

| Pregunta | Traducción |
|----------|------------|
| Which branch should I merge into? | ¿A cuál rama debería hacer merge? |
| Which version of Node are you running? | ¿Qué versión de Node estás corriendo? |
| Which database do you recommend? | ¿Cuál base de datos recomiendas? |
| Which framework is better for this project? | ¿Cuál framework es mejor para este proyecto? |
| Which endpoint should I use? | ¿Cuál endpoint debería usar? |

> **What vs Which:** Usa **which** cuando las opciones son limitadas y conocidas.
> "Which language: Python or Go?" vs "What language do you prefer?"

---

## 10. Preguntas frecuentes en el trabajo diario

```text
What's blocking you?                 - ¿Qué te bloquea?
Where can I find the documentation?  - ¿Dónde puedo encontrar la documentación?
When was this last updated?          - ¿Cuándo fue la última actualización?
Why is this hardcoded?               - ¿Por qué está esto hardcodeado?
How do I set up the dev environment? - ¿Cómo configuro el entorno de desarrollo?
Who owns this repository?            - ¿Quién es dueño de este repositorio?
Which PR fixes this issue?           - ¿Cuál PR arregla este issue?
```

---

## 11. Errores comunes

| Incorrecto | Correcto | Explicación |
|------------|----------|-------------|
| What means this error? | What does this error mean? | Falta el auxiliar "does" |
| Where I can find it? | Where can I find it? | El auxiliar va antes del sujeto |
| Why the server is down? | Why is the server down? | Se invierte sujeto y verbo |
| How it works? | How does it work? | Falta el auxiliar "does" |
| Who did wrote this? | Who wrote this? | "Who" como sujeto no necesita auxiliar |
