# Ejercicios: Questions & Negations

## Practica preguntas y negaciones en contexto de programación

---

## Ejercicio 1: Forma la pregunta correcta

Convierte estas oraciones en preguntas. Ejemplo:
- The server runs on port 3000. -> **Does the server run on port 3000?**

1. The API returns a 404 error.
2. You have access to the staging server.
3. The tests are passing.
4. She uses TypeScript for the frontend.
5. The database is connected.
6. They deploy every week.
7. The build takes more than 10 minutes.
8. He can fix the bug today.

---

## Ejercicio 2: Respuestas cortas

Responde con la forma corta apropiada (afirmativa o negativa según se indique).

1. Does the API support GraphQL? (No) -> _______________
2. Is the server running? (Yes) -> _______________
3. Do you use Docker? (Yes) -> _______________
4. Are the tests passing? (No) -> _______________
5. Can you review my PR? (Yes) -> _______________
6. Has the deploy finished? (No) -> _______________
7. Will it be ready by Monday? (Yes) -> _______________
8. Are you blocked? (No) -> _______________

---

## Ejercicio 3: Elige la WH word correcta

Completa con: **What / Where / When / Why / How / Who / Which**

1. _______ does this function do?
2. _______ is the config file located?
3. _______ is the build failing?
4. _______ do I install the dependencies?
5. _______ branch should I merge into?
6. _______ wrote this code?
7. _______ is the deadline for this feature?
8. _______ version of Python are you using?
9. _______ many endpoints does the API have?
10. _______ is responsible for the database?

---

## Ejercicio 4: Corrige los errores

Cada oración tiene un error. Encuéntralo y corrígelo.

1. Does the server is running?
2. What means this error?
3. I don't can access the database.
4. Why the build is failing?
5. It doesn't works on my machine.
6. Where I can find the logs?
7. Who did wrote this function?
8. He don't have access.
9. Yes, it's. (como respuesta corta)
10. I don't have nothing to report.

---

## Ejercicio 5: Convierte a negativo

Transforma cada oración a su forma negativa usando contracciones.

1. The test passes on CI.
2. I understand the error.
3. The server is responding.
4. We can deploy today.
5. It will compile without errors.
6. She has reviewed the PR.
7. They are ready to merge.
8. The function returns a value.

---

## Ejercicio 6: Escribe la pregunta para cada respuesta

Lee la respuesta y escribe la pregunta que la originó.

1. Pregunta: _________________ -> It uses port 8080.
2. Pregunta: _________________ -> The logs are in the /var/log directory.
3. Pregunta: _________________ -> Because the dependency is outdated.
4. Pregunta: _________________ -> You run `npm install` in the terminal.
5. Pregunta: _________________ -> Maria wrote that module.
6. Pregunta: _________________ -> The deadline is next Friday.

---

## Ejercicio 7: Traduce al inglés

1. ¿La API devuelve JSON?
2. No puedo reproducir el bug.
3. ¿Dónde está definida esta función?
4. El servidor no está respondiendo.
5. ¿Por qué falla el test?
6. No deberíamos hacer deploy el viernes.
7. ¿Quién revisó este PR?
8. No funciona en mi máquina.
9. ¿Cuánto tiempo tarda el build?
10. Nadie ha respondido al issue todavía.

---

## Ejercicio 8: Situación real — Daily Standup

Escribe en inglés las preguntas y negaciones que usarías en estas situaciones:

1. Quieres saber si el servidor de staging está funcionando.
2. Necesitas preguntar quién es responsable del módulo de pagos.
3. Quieres reportar que no pudiste acceder a la base de datos.
4. Necesitas saber cuándo estará listo el nuevo endpoint.
5. Quieres preguntar por qué el pipeline de CI está fallando.
