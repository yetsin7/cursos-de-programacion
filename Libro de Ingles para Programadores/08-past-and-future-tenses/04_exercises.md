## Ejercicio 3: Pasado simple vs. presente perfecto

Elige la forma correcta.

1. I ______ (fixed / have fixed) the bug. It works now.
2. I ______ (fixed / have fixed) the bug yesterday at 3 PM.
3. ______ you ever ______ (Did...use / Have...used) Kubernetes?
4. She ______ (deployed / has deployed) the hotfix last night.
5. We ______ (didn't finish / haven't finished) the migration yet.
6. The server ______ (crashed / has crashed) three times this week.
7. I ______ (worked / have worked) here for two years.
8. They ______ (released / have released) v2.0 in March 2025.

---

## Ejercicio 4: Completa con presente perfecto

Usa have/has + participio pasado.

1. I ______ (just / finish) the code review.
2. She ______ (not / push) her changes yet.
3. We ______ (already / migrate) to the new server.
4. ______ you ______ (ever / work) with GraphQL?
5. The build ______ (fail) twice today.
6. They ______ (never / have) a production outage.
7. I ______ (not / test) this on mobile yet.
8. He ______ (write) over 300 unit tests.

---

## Ejercicio 5: Elige el futuro correcto

Completa con **will**, **going to** o **presente continuo**.

1. [Decisión inmediata] "There's a bug? I ______ take a look."
2. [Plan decidido] "We ______ refactor the payment module next sprint."
3. [Evento agendado] "The retrospective ______ at 4 PM."
4. [Predicción] "This approach ______ cause problems in production."
5. [Intención] "I ______ learn Docker this quarter."
6. [Ya confirmado] "She ______ at the conference next Tuesday."
7. [Promesa] "Don't worry, I ______ the PR ready by tomorrow."
8. [Predicción con evidencia] "Look at the CPU usage — the server ______ crash."

## Ejercicio 6: Forma oraciones en futuro

Escribe la oración usando la forma de futuro indicada.

1. (will) I / review / your PR / later
2. (going to) We / migrate / to PostgreSQL / next month
3. (present continuous) The team / have a meeting / at 2 PM
4. (will have) By Friday / I / finish / the module
5. (will be + ing) Tomorrow / she / work on / the API
6. (won't) The feature / be ready / for this sprint

## Ejercicio 7: Traduce al inglés

### Pasado simple
1. Arreglé el bug ayer por la mañana.
2. El servidor se cayó a las 2 AM.
3. Ella escribió todos los tests unitarios.
4. No encontramos ningún problema de seguridad.
5. ¿Desplegaste los cambios?

### Presente perfecto
6. Ya he terminado el code review.
7. Ella no ha subido sus cambios todavía.
8. ¿Alguna vez has usado Rust?
9. El build ha fallado tres veces hoy.
10. Hemos trabajado en este proyecto desde enero.

### Futuro
11. Voy a refactorizar este módulo la próxima semana.
12. Lo arreglaré ahora mismo.
13. La reunión empieza a las 3 PM.
14. Para el viernes habré terminado la migración.
15. Esto no va a funcionar en producción.

## Ejercicio 8: Situación — Standup completo

Escribe un standup completo en inglés usando los tres tiempos:

**Ayer (pasado simple):** Trabajaste en el módulo de autenticación y arreglaste dos bugs.

**Estado actual (presente perfecto):** Ya has creado un PR pero no has recibido review todavía.

**Hoy (futuro):** Vas a trabajar en el módulo de pagos y tienes una reunión a las 2 PM.

Escribe tu standup:

```text
Yesterday: _______________________________________________
_______________________________________________

Current status: _______________________________________________

Today: _______________________________________________
_______________________________________________
```
