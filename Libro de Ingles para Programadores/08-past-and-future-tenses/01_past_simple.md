# Past Simple

## Pasado simple en contexto de programación

El pasado simple se usa para hablar de acciones completadas en un momento específico del pasado. Es el tiempo verbal que más usarás en standups ("Yesterday I fixed..."), retrospectivas y reportes de bugs.

---

## 1. Verbos regulares: agregar -ed

La mayoría de los verbos forman el pasado agregando **-ed** al final.

### Reglas de ortografía

| Regla | Base | Pasado | Ejemplo |
|-------|------|--------|---------|
| Agregar -ed | deploy | deployed | I deployed the fix yesterday. |
| Termina en -e: agregar -d | configure | configured | She configured the server. |
| Consonante + y: cambiar y por -ied | copy | copied | I copied the file to staging. |
| CVC corto: doblar consonante | stop | stopped | The process stopped unexpectedly. |

## 3. Estructura del pasado simple

### Afirmativa

```
Sujeto + verbo en pasado + complemento
```

```text
I fixed the bug yesterday.
We deployed the new version last night.
The server crashed at 3 AM.
She wrote all the unit tests.
```

### Negativa: didn't + verbo base

```
Sujeto + didn't + verbo base + complemento
```

```text
I didn't push to main directly.
The test didn't pass on the first try.
We didn't find any security issues.
She didn't merge without approval.
```

> **Cuidado:** Después de **didn't**, el verbo va en forma base.
> Incorrecto: *I didn't fixed the bug.*
> Correcto: **I didn't fix the bug.**

### Pregunta: Did + sujeto + verbo base?

```text
Did you run the tests?
Did the deploy succeed?
Did she review the PR?
Did they update the documentation?
```

## 4. Expresiones de tiempo con pasado simple

| Expresión | Significado | Ejemplo |
|-----------|-------------|---------|
| yesterday | ayer | I fixed it yesterday. |
| last week/month/year | la semana/mes/año pasado | We deployed last Friday. |
| two days ago | hace dos días | The bug appeared two days ago. |
| this morning | esta mañana | I merged the PR this morning. |
| earlier today | más temprano hoy | She pushed the fix earlier today. |
| in 2024 | en 2024 | We migrated to Kubernetes in 2024. |

## 5. Ejemplos en el día a día del desarrollador

### En el daily standup

```text
Yesterday I fixed the login bug and deployed the hotfix.
I worked on the payment module all day.
I ran into an issue with the database connection.
I created a PR for the new feature.
I spent most of the day debugging the API.
```

### En reportes de incidentes

```text
The server went down at 2:15 AM.
We identified the root cause within 30 minutes.
The team deployed a hotfix at 3:00 AM.
The issue affected approximately 500 users.
We restored full service by 3:30 AM.
```
