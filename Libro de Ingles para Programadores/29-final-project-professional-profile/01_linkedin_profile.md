# Lesson 01: Writing a Professional LinkedIn Profile

## Introducción

LinkedIn es la plataforma mas importante para conseguir trabajo en tecnología a nivel internacional. Un perfil bien escrito en ingles puede abrirte puertas a oportunidades remotas y en el extranjero. Cada sección tiene su propio formato y vocabulario que debes dominar.

---

## La formula del Headline

Tu headline es lo primero que ven los reclutadores. Tiene un máximo de 220 carácteres.

### Formulas efectivas

| Formula | Ejemplo |
|---------|---------|
| **Rol + Tecnología** | "Senior Flutter Developer \| Mobile Apps" |
| **Rol + Empresa** | "Full-Stack Developer at Google" |
| **Rol + Especialidad** | "Backend Engineer \| Distributed Systems" |
| **Rol + Valor** | "Mobile Developer \| Building apps used by 1M+ users" |
| **Rol + Stack** | "Frontend Developer \| React, TypeScript, Next.js" |

### Buenos headlines

```
"Senior Mobile Developer | Flutter & Dart | Building 
offline-first apps for emerging markets"

"Full-Stack Engineer | React + Node.js | Passionate about 
clean architecture and developer experience"

"Software Engineer | Python, Go | Scaling systems that 
serve millions of requests per day"
```

### Headlines que debes evitar

| Malo | Por que | Mejor |
|------|---------|-------|
| "Looking for opportunities" | Suena desesperado | Describe lo que haces |
| "Unemployed developer" | Negativo | "Software Engineer \| Open to new challenges" |
| "Code ninja rockstar" | No profesional | "Senior Developer \| React & TypeScript" |
| Solo el titulo del puesto | Muy generico | Agrega tecnologías o valor |

## About / Summary Section

Esta sección es tu pitch personal. Debe tener entre 3-5 parrafos cortos.

### Estructura recomendada

```
Paragraph 1: Who you are and what you do
Paragraph 2: Your key skills and technologies
Paragraph 3: What drives you / your passion
Paragraph 4: What you're looking for (optional)
```

### Ejemplo completo

```
I'm a software engineer with 5+ years of experience building 
mobile applications with Flutter and cross-platform solutions 
that prioritize performance and user experience.

My expertise spans the full mobile development lifecycle: from 
architecture design and state management to CI/CD pipelines and 
app store deployment. I specialize in offline-first apps, clean 
architecture, and scalable codebases using Dart, Firebase, and 
REST APIs.

I'm passionate about writing clean, maintainable code and 
mentoring junior developers. I believe great software comes 
from combining solid engineering practices with a deep 
understanding of user needs.

Currently open to remote opportunities where I can contribute 
to impactful mobile products.
```

### Vocabulario para el Summary

| Frase | Uso |
|-------|-----|
| "I'm a [role] with X+ years of experience in..." | Presentación |
| "I specialize in..." | Tu especialidad |
| "My expertise spans..." | Rango de habilidades |
| "I'm passionate about..." | Lo que te motiva |
| "I thrive in environments where..." | Donde trabajas mejor |
| "I have a proven track record of..." | Resultados demostrables |
| "Currently open to..." | Lo que buscas |
| "I bring a strong background in..." | Lo que aportas |

## Experience Section

La sección de experiencia es la mas importante. Usa **action verbs** y **metricas** para describir tus logros.

### Formula: Action Verb + What + Result/Impact

```
Developed a real-time notification system that reduced 
user response time by 35%.

Led a team of 4 developers to deliver a mobile banking 
app serving 50,000+ daily active users.

Reduced API response time from 2s to 200ms by implementing 
Redis caching and query optimization.

Architected a microservices system that handles 10M+ 
requests per day with 99.9% uptime.
```

### Action verbs por categoría

| Categoría | Verbos |
|-----------|--------|
| **Construir** | Developed, Built, Created, Designed, Implemented, Engineered |
| **Liderar** | Led, Managed, Mentored, Coordinated, Directed, Supervised |
| **Mejorar** | Optimized, Improved, Enhanced, Streamlined, Refactored, Upgraded |
| **Entregar** | Delivered, Shipped, Launched, Released, Deployed, Published |
| **Resolver** | Fixed, Resolved, Debugged, Troubleshot, Diagnosed, Eliminated |
| **Automatizar** | Automated, Scripted, Configured, Integrated, Orchestrated |
| **Escalar** | Scaled, Expanded, Grew, Extended, Migrated, Modernized |

### Ejemplo de Experience Entry

```
Senior Mobile Developer
TechCorp | Jan 2022 - Present | Remote

- Developed and maintained a Flutter mobile app with 100K+ 
  downloads on Google Play and App Store
- Led the migration from Provider to Riverpod, reducing 
  boilerplate code by 40% and improving testability
- Implemented offline-first architecture using Hive and 
  background sync, enabling usage in low-connectivity areas
- Mentored 3 junior developers through code reviews and 
  pair programming sessions
- Reduced app crash rate from 2.1% to 0.3% by implementing 
  comprehensive error handling and monitoring
```

### Metricas que impresionan

| Tipo de metrica | Ejemplo |
|----------------|---------|
| **Usuarios** | "...serving 50K+ daily active users" |
| **Rendimiento** | "...reducing load time by 60%" |
| **Equipo** | "...leading a team of 5 engineers" |
| **Escala** | "...processing 1M+ transactions daily" |
| **Calidad** | "...achieving 95% test coverage" |
| **Velocidad** | "...delivering 2 weeks ahead of schedule" |
| **Ahorro** | "...saving $10K/month in infrastructure costs" |

## Skills Section

### Organizacion recomendada

| Prioridad | Tipo de skills |
|-----------|---------------|
| **Alta** | Tus tecnologías principales (Flutter, React, Python) |
| **Media** | Tecnologías secundarias (Docker, AWS, PostgreSQL) |
| **Baja** | Soft skills (no las pongas si puedes llenar con tech) |

### Skills que los reclutadores buscan

```
Languages: Dart, JavaScript, TypeScript, Python, Go
Frameworks: Flutter, React, Next.js, Node.js, Django
Databases: PostgreSQL, MongoDB, Redis, Firebase
Tools: Git, Docker, Kubernetes, CI/CD, AWS
Practices: Clean Architecture, TDD, Agile/Scrum, Code Review
```

## Recommendations

### Como pedir una recomendación

```
"Hi [Name], I really enjoyed working with you at [Company]. 
Would you be willing to write a brief LinkedIn recommendation 
about our collaboration on [Project]? I'd be happy to write 
one for you as well."
```

### Como escribir una recomendación

```
"I had the pleasure of working with [Name] for two years 
at [Company]. As a mobile developer, [Name] consistently 
delivered high-quality code and was always willing to help 
teammates. Their deep expertise in Flutter and clean 
architecture significantly improved our codebase quality. 
I highly recommend [Name] for any mobile development role."
```

## Vocabulario esencial de LinkedIn

| Término | Significado |
|---------|-------------|
| **Headline** | Titulo debajo de tu nombre |
| **Open to work** | Disponible para trabajar |
| **Hiring** | Contratando |
| **Endorsement** | Validación de una skill |
| **Recommendation** | Recomendación escrita |
| **Connection** | Contacto en LinkedIn |
| **InMail** | Mensaje directo de LinkedIn |
| **Recruiter** | Reclutador |
| **Hiring manager** | Gerente de contratacion |
| **Remote-friendly** | Acepta trabajo remoto |

---

## Resumen

- Tu headline debe comunicar tu rol, tecnologías y valor en menos de 220 carácteres
- El summary cuenta tu historia en 3-5 parrafos: quien eres, que haces, que buscas
- En experience, usa action verbs + metricas para describir logros, no responsabilidades
- Las skills deben priorizar tecnologías sobre soft skills
- Pide recomendaciones a companeros con los que trabajaste directamente
