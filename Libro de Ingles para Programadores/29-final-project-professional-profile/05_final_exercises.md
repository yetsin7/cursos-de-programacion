## Checklist de revision

Antes de considerar tu proyecto completo, verifica:

### LinkedIn
- [ ] El headline tiene menos de 220 caracteres e incluye tu rol y tecnologias
- [ ] El summary tiene 3-4 parrafos bien estructurados
- [ ] La experiencia usa action verbs y metricas
- [ ] No hay errores gramaticales ni de ortografia

### GitHub
- [ ] El profile README tiene: About, Tech Stack, Projects, Contact
- [ ] Las descripciones de repos son claras y concisas
- [ ] Al menos un repo tiene un README profesional completo

### Resume
- [ ] Es de 1-2 paginas
- [ ] No incluye foto, edad ni estado civil
- [ ] Cada bullet point tiene: action verb + what + result
- [ ] Incluye metricas cuantificables
- [ ] Las keywords coinciden con el puesto deseado

### Cover Letter
- [ ] Tiene 250-400 palabras
- [ ] Menciona la empresa y el puesto especificamente
- [ ] Conecta tu experiencia con las necesidades del puesto
- [ ] Tiene un cierre con call to action
- [ ] Es diferente al resume (cuenta una historia)

### General
- [ ] Todo esta escrito en ingles correcto
- [ ] El tono es profesional pero con personalidad
- [ ] No hay traducciones literales del espanol
- [ ] Los action verbs son variados (no repetir "developed" en todo)
- [ ] Las metricas son realistas y verificables
