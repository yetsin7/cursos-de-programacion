# Lesson 01: Native-Level Vocabulary

## Introduccion

Cuando escuchas a programadores nativos hablar en ingles, usan un vocabulario que no encontraras en los diccionarios tradicionales. Son expresiones del mundo tech que tienen significados especificos en contexto. Dominarlas te hara sonar profesional y entender conversaciones reales.

---

## Vocabulario de negocios y estrategia tech

| Termino | Pronunciacion | Significado | Ejemplo |
|---------|---------------|-------------|---------|
| **leverage** | LE-ver-ij | Aprovechar, usar a tu favor | "We can leverage the existing API." |
| **streamline** | STRIM-lain | Simplificar, hacer mas eficiente | "Let's streamline the deployment process." |
| **onboard** | ON-bord | Integrar a alguien nuevo | "We need to onboard three new developers." |
| **offboard** | OF-bord | Proceso de salida de un empleado | "Don't forget to offboard their access." |
| **ramp up** | RAMP ap | Aumentar gradualmente, ponerse al dia | "It takes a month to ramp up on this codebase." |
| **bandwidth** | BAND-width | Capacidad disponible (de una persona) | "I don't have bandwidth for that this sprint." |
| **deep dive** | DIP daiv | Analisis profundo | "Let's do a deep dive into the performance issues." |
| **north star** | NORTH star | Objetivo principal, vision guia | "User satisfaction is our north star metric." |
| **low-hanging fruit** | LOU-janging frut | Tarea facil con alto impacto | "Let's tackle the low-hanging fruit first." |
| **move the needle** | MUV de NI-dol | Hacer un impacto significativo | "Will this feature really move the needle?" |

## Vocabulario de proyectos y desarrollo

| Termino | Significado | Ejemplo |
|---------|-------------|---------|
| **greenfield** | Proyecto nuevo desde cero | "This is a greenfield project — no legacy code." |
| **brownfield** | Proyecto existente con codigo legacy | "We're working on a brownfield codebase." |
| **evergreen** | Que se mantiene actualizado automaticamente | "We use evergreen browsers as our baseline." |
| **pain point** | Problema que causa frustracion | "Slow deploys are a major pain point." |
| **bottleneck** | Cuello de botella, lo que limita el rendimiento | "The database is our bottleneck." |
| **blocker** | Algo que impide avanzar | "I have a blocker — the API is down." |
| **showstopper** | Problema tan grave que detiene todo | "This memory leak is a showstopper." |
| **dealbreaker** | Condicion que hace inaceptable algo | "No TypeScript support is a dealbreaker for us." |
| **workaround** | Solucion temporal | "There's a workaround until the fix is released." |
| **tech debt** | Deuda tecnica | "We need to address our growing tech debt." |
| **scope creep** | Expansion no controlada del alcance | "Watch out for scope creep in this project." |

## Vocabulario de codigo y arquitectura

| Termino | Significado | Ejemplo |
|---------|-------------|---------|
| **hack** (positivo) | Solucion ingeniosa y rapida | "Nice hack to get around the API limit!" |
| **hack** (negativo) | Solucion sucia, parche temporal | "This is a terrible hack. We need a real fix." |
| **edge case** | Caso limite o poco comun | "We need to handle this edge case." |
| **happy path** | Flujo ideal donde todo funciona | "The happy path works, but errors aren't handled." |
| **golden path** | La ruta recomendada/estandar | "Follow the golden path for setup." |
| **blast radius** | Alcance del impacto de un cambio o fallo | "What's the blast radius if this service goes down?" |
| **escape hatch** | Mecanismo para saltarse restricciones | "useRef is React's escape hatch for DOM access." |
| **guardrails** | Limites de seguridad que previenen errores | "TypeScript adds guardrails to your code." |
| **footgun** | Feature que facilita hacerte dano a ti mismo | "eval() is a footgun — avoid it." |
| **bikeshedding** | Discutir detalles triviales excesivamente | "Stop bikeshedding about variable names." |

## Vocabulario de personas y equipos

| Termino | Significado | Ejemplo |
|---------|-------------|---------|
| **stakeholder** | Parte interesada | "The stakeholders want a demo on Friday." |
| **IC** (Individual Contributor) | Contribuidor individual (no manager) | "She's a senior IC, not a manager." |
| **DRI** (Directly Responsible Individual) | Persona directamente responsable | "Who's the DRI for this project?" |
| **bus factor** | Cuantas personas pueden ser atropelladas por un bus antes de que el proyecto muera | "Our bus factor is 1 — only John knows this code." |
| **tribal knowledge** | Conocimiento no documentado que solo algunos tienen | "Too much tribal knowledge on this team." |
| **rubber duck debugging** | Explicar el problema en voz alta para resolverlo | "Try rubber duck debugging — explain it to me." |
| **yak shaving** | Hacer tareas aparentemente no relacionadas para lograr tu objetivo | "I spent all day yak shaving instead of coding." |

## Expresiones de estado y progreso

| Expresion | Significado | Contexto |
|-----------|-------------|----------|
| **ship it** | Publicalo, lancalo | "It's good enough. Ship it!" |
| **nuke it** | Borralo completamente | "This module is beyond repair. Nuke it." |
| **punt** | Postergar, patear para adelante | "Let's punt on this until next sprint." |
| **spike** | Investigacion tecnica exploratoria | "Let's do a spike on GraphQL before committing." |
| **dog-fooding** | Usar tu propio producto internamente | "We're dog-fooding the new API." |
| **bike shed** | Debatir cosas triviales | "We spent an hour bikeshedding the color of the button." |
| **rubber stamp** | Aprobar sin revisar realmente | "Don't just rubber stamp the PR — review it." |
| **over-engineer** | Complicar mas de lo necesario | "Don't over-engineer this. A simple solution is fine." |
| **boil the ocean** | Intentar resolver todo de golpe | "Let's not boil the ocean. Start small." |
| **eat your own dog food** | Usa lo que tu construyes | "If we don't eat our own dog food, why would users?" |

## Expresiones de calidad y rendimiento

| Expresion | Significado |
|-----------|-------------|
| **bullet-proof** | Extremadamente robusto |
| **rock-solid** | Muy estable y confiable |
| **blazing fast** | Extremadamente rapido |
| **battle-tested** | Probado en produccion real |
| **production-ready** | Listo para produccion |
| **half-baked** | Incompleto, no terminado bien |
| **spaghetti code** | Codigo enredado y dificil de mantener |
| **clean code** | Codigo limpio, legible y bien organizado |
| **brittle** | Fragil, se rompe facilmente |
| **flaky** | Inestable, a veces funciona y a veces no |

## Acronimos informales del dia a dia

| Acronimo | Significado | Uso |
|----------|-------------|-----|
| **LGTM** | Looks Good To Me | Aprobacion en code review |
| **WIP** | Work In Progress | PR o tarea en progreso |
| **PTAL** | Please Take A Look | Pedir revision |
| **AFAIK** | As Far As I Know | Hasta donde yo se |
| **IMO / IMHO** | In My (Humble) Opinion | En mi opinion |
| **FWIW** | For What It's Worth | Por si sirve de algo |
| **TL;DR** | Too Long; Didn't Read | Resumen de algo largo |
| **YMMV** | Your Mileage May Vary | Los resultados pueden variar |
| **RTFM** | Read The F***ing Manual | Lee la documentacion |
| **WDYT** | What Do You Think? | Que opinas? |
| **SGTM** | Sounds Good To Me | Me parece bien |
| **NIT** | Nitpick (detalle menor) | Comentario menor en code review |

## Usando el vocabulario en contexto

### En una reunion de sprint

```
"We have some low-hanging fruit in the backlog that could 
move the needle on performance. I'd suggest we tackle those 
first, then deep dive into the authentication blocker. I 
don't have bandwidth for the greenfield project this sprint, 
so let's punt on that."
```

### En un code review

```
"LGTM overall! A few nits:
- This is a bit over-engineered for what we need. A simpler 
  approach would work.
- Watch out for this edge case — what happens when the input 
  is null?
- The happy path looks good, but the error handling is 
  half-baked."
```

### En Slack

```
"Hey team, heads up: the deploy is blocked because of a 
flaky test in CI. I'm looking into it but the blast radius 
is small — only affects the settings page. There's a 
workaround: skip the test and we can ship it today. WDYT?"
```

---

## Resumen

- El vocabulario nativo tech va mas alla de los diccionarios tradicionales
- Expresiones como "low-hanging fruit", "blast radius" y "guardrails" son de uso diario
- Conocer estos terminos te permite participar en reuniones y code reviews con confianza
- Los acronimos informales (LGTM, PTAL, TL;DR) aceleran la comunicacion escrita
- Practica usando estas expresiones en contextos reales para internalizarlas
