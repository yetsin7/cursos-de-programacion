# Ejercicios Integrales - Capítulo 03: Pronombres y Artículos

## Introducción

Estos ejercicios combinan todos los temas del capítulo: pronombres personales (sujeto y objeto), artículos (a/an/the/cero) y demostrativos y posesivos.

---

## Sección A: Pronombres personales

### Ejercicio A1: Elige el pronombre de sujeto correcto

1. ___ is the most popular framework right now. (It / Him)
2. ___ always write documentation for my functions. (Me / I)
3. ___ merged the branch yesterday. (She / Her)
4. ___ are responsible for the deployment. (They / Them)
5. ___ need to fix the CI pipeline. (Us / We)

### Ejercicio A2: Elige el pronombre de objeto correcto

1. The manager assigned the task to ___. (I / me)
2. I reviewed the code and approved ___. (it / its)
3. Can you send ___ the API documentation? (he / him)
4. The function processes ___ and returns a result. (they / them)
5. She explained the architecture to ___. (we / us)

### Ejercicio A3: Reemplaza el sustantivo subrayado por un pronombre

1. **The server** is down. → ___ is down.
2. I need to talk to **María**. → I need to talk to ___.
3. **The tests** are failing. → ___ are failing.
4. Send the report to **Carlos**. → Send the report to ___.
5. **Ana and I** finished the sprint. → ___ finished the sprint.

## Sección C: Demostrativos y posesivos

### Ejercicio C1: this / that / these / those

1. ___ commit I just made has a typo. (cerca)
2. ___ tests on the staging server failed. (lejos)
3. Look at ___ error messages. (cerca, plural)
4. ___ approach you suggested last week worked. (lejos)
5. ___ is the new database schema. (cerca)

### Ejercicio C2: Posesivos (my / your / his / her / its / our / their)

1. The class inherits from ___ parent class. (de la clase)
2. I forgot to push ___ changes. (yo)
3. ___ team completed the sprint early. (nosotros)
4. She reviewed ___ colleague's code. (de ella)
5. They updated ___ CI/CD pipeline. (de ellos)
6. Check ___ inbox for the meeting invite. (tú)

### Ejercicio C3: its vs. it's

1. ___ the third time the build has failed.
2. The API and ___ rate limits are documented.
3. ___ returning a 404 error.
4. Every framework has ___ strengths and weaknesses.
5. ___ important to version ___ dependencies.

## Sección D: Ejercicio mixto completo

### Completa cada oración con la palabra correcta

1. ___ (I/Me) created ___ (a/an) API endpoint for ___ (the/---) user authentication.
2. ___ (This/These) functions return ___ (a/an) object with ___ (it's/its) metadata.
3. Can you send ___ (I/me) ___ (the/a) error log from ___ (your/you're) server?
4. ___ (She/Her) reviewed ___ (my/mine) code and found ___ (a/an) issue in ___ (the/a) validation logic.
5. ___ (They/Them) deployed ___ (their/they're) application to ___ (the/---) AWS.
6. ___ (That/Those) endpoints ___ (is/are) deprecated. Use ___ (this/these) new ones instead.
7. ___ (The/---) Python is ___ (a/an) interpreted language, and ___ (it's/its) syntax is clean.
8. ___ (We/Us) need to update ___ (our/ours) dependencies before ___ (the/a) release.

---

## Sección E: Traduce al inglés

1. Ella nos envió las credenciales del servidor.
2. Este es un error conocido en la documentación.
3. Python es un lenguaje de programación. Es muy popular.
4. Esos métodos son privados. No puedes acceder a ellos.
5. Mi código está listo. ¿Está listo el tuyo?
6. La API y sus endpoints están documentados.
7. Necesito hablar con él sobre el deployment.
8. Estas son las nuevas dependencias del proyecto.
9. Hay un bug. Necesitamos arreglarlo antes del release.
10. Nuestro equipo usa React para el frontend.

## Sección F: Encuentra y corrige los errores

Cada oración tiene uno o más errores. Encuéntralos y corrígelos.

1. "Me wrote a function that returns the boolean."
2. "The JavaScript is a popular language."
3. "This variables need to be initialized."
4. "Check it's documentation before using the library."
5. "Her sent the code to he for review."
6. "I am the developer. The my code is clean."
7. "That endpoints are deprecated. Use this ones."
8. "The function lost it's value and returns the null."
