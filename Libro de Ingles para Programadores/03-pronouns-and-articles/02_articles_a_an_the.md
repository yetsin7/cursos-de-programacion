# Artículos: A, An, The (y el artículo cero)

## Introducción

Los artículos son una de las mayores dificultades para hispanohablantes. En español tenemos "un/una/unos/unas" y "el/la/los/las". En inglés solo hay tres opciones: **a**, **an** y **the**, más la opción de **no usar artículo** (artículo cero). Las reglas son diferentes al español.

---

## Artículo indefinido: A / An (un, una)

Se usan con sustantivos **singulares contables** cuando se mencionan por primera vez o de forma general.

### Regla: A vs. An

| Regla | Artículo | Ejemplo |
|-------|----------|---------|
| Antes de sonido **consonante** | **a** | a function, a server, a bug |
| Antes de sonido **vocal** | **an** | an array, an error, an API |

> **Clave:** Depende del **sonido**, no de la letra. "A user" (suena /yúser/, consonante). "An hour" (suena /áuer/, vocal).

### Ejemplos en programación

| Con "a" | Con "an" |
|---------|----------|
| a function | an array |
| a variable | an error |
| a server | an API |
| a database | an object |
| a bug | an integer |
| a component | an endpoint |
| a framework | an instance |
| a repository | an interface |
| a string | an element |
| a boolean | an update |
| a parameter | an exception |
| a module | an HTTP request |

### Casos especiales por sonido

| Palabra | Artículo | Razón |
|---------|----------|-------|
| a **U**RL | a | "URL" suena /yú-ar-el/ (consonante) |
| a **u**ser | a | "user" suena /yúser/ (consonante) |
| a **u**nix system | a | "unix" suena /yúnix/ (consonante) |
| an **S**QL query | an | "SQL" suena /es-kiú-el/ (vocal) |
| an **H**TML file | an | "HTML" suena /eich-tí-em-el/ (vocal) |
| an **h**our | an | "hour" es muda, suena /áuer/ (vocal) |

## Artículo definido: The (el, la, los, las)

Se usa cuando hablamos de algo **específico** o ya mencionado.

### Cuándo usar "the"

| Situación | Ejemplo | Español |
|-----------|---------|---------|
| Ya se mencionó antes | I found a bug. **The** bug is in the login module. | Encontré un bug. **El** bug está en el módulo de login. |
| Es único o específico | **The** database is down | **La** base de datos está caída |
| Ambos saben de qué hablan | Can you review **the** PR? | ¿Puedes revisar **el** PR? |
| Hay solo uno | **The** CEO approved the project | **El** CEO aprobó el proyecto |
| Con superlativos | **The** fastest algorithm | **El** algoritmo más rápido |

### Ejemplos en programación

| Inglés | Español |
|--------|---------|
| **The** database stores user data | **La** base de datos almacena datos del usuario |
| **The** function returns a promise | **La** función retorna una promesa |
| **The** server is running on port 3000 | **El** servidor está corriendo en el puerto 3000 |
| Check **the** error log | Revisa **el** log de errores |
| **The** deployment failed | **El** despliegue falló |
| **The** tests are in the /tests folder | **Los** tests están en la carpeta /tests |
| Update **the** dependencies | Actualiza **las** dependencias |

## Artículo cero: Sin artículo

En inglés hay muchos casos donde **no se usa artículo**, aunque en español sí se use. Esto es muy relevante en tecnología.

### 1. Nombres de tecnologías, lenguajes y herramientas

| Correcto | Incorrecto |
|----------|------------|
| I use **JavaScript** | ~~I use the JavaScript~~ |
| **Python** is popular | ~~The Python is popular~~ |
| Install **Docker** | ~~Install the Docker~~ |
| We migrated to **React** | ~~We migrated to the React~~ |
| **Git** is essential | ~~The Git is essential~~ |
| **Linux** is open source | ~~The Linux is open source~~ |

### 2. Sustantivos plurales o incontables en sentido general

| Correcto | Incorrecto |
|----------|------------|
| **Developers** use Git | ~~The developers use Git~~ (en general) |
| **Software** is complex | ~~The software is complex~~ (en general) |
| **Code** should be clean | ~~The code should be clean~~ (en general) |
| **Bugs** are inevitable | ~~The bugs are inevitable~~ (en general) |

> **Nota:** Si hablas de algo específico, sí se usa "the": "The developers **on our team** use Git."

### 3. Conceptos abstractos y disciplinas

| Correcto | Incorrecto |
|----------|------------|
| I study **computer science** | ~~I study the computer science~~ |
| **Programming** is fun | ~~The programming is fun~~ |
| She's good at **debugging** | ~~She's good at the debugging~~ |

---

## Errores comunes de hispanohablantes

| Error | Correcto | Explicación |
|-------|----------|-------------|
| ~~I use the React~~ | I use React | Sin artículo para nombres de tecnología |
| ~~The JavaScript is popular~~ | JavaScript is popular | Sin artículo para lenguajes |
| ~~I found bug~~ | I found **a** bug | Necesita "a" (singular contable primera mención) |
| ~~I am developer~~ | I am **a** developer | Necesita "a" con profesiones |
| ~~She is good the programmer~~ | She is a good programmer | Artículo indefinido, no definido |
| ~~Install the Node.js~~ | Install Node.js | Sin artículo para herramientas |
| ~~The code is in the Python~~ | The code is in Python | Sin artículo para lenguajes |

---

## Guía rápida de decisión

```
¿Es un nombre propio de tecnología? (React, Python, Docker)
  → SIN artículo

¿Es singular contable, mencionado por primera vez?
  → A / AN

¿Ya fue mencionado o es algo específico conocido?
  → THE

¿Es plural o incontable en sentido general?
  → SIN artículo

¿Empieza con sonido vocal?
  → AN (an array, an error)

¿Empieza con sonido consonante?
  → A (a function, a bug)
```

## Ejercicios

### Ejercicio 1: Completa con a / an

1. ___ API endpoint
2. ___ boolean value
3. ___ HTTP request
4. ___ user interface
5. ___ SQL database
6. ___ unique identifier
7. ___ open-source project
8. ___ URL parameter

### Ejercicio 2: Completa con the o deja vacío (---)

1. ___ Python is a versatile language.
2. I need to check ___ logs from yesterday.
3. ___ Docker simplifies deployment.
4. ___ server we use is in AWS.
5. ___ software engineering requires patience.
6. She's ___ best developer on the team.
7. We use ___ React for the frontend.
8. ___ database crashed this morning.

### Ejercicio 3: Elige a / an / the / (nada)

1. I am ___ full-stack developer.
2. ___ JavaScript is not ___ same as Java.
3. There is ___ error in ___ code.
4. ___ function returns ___ object.
5. Install ___ Node.js before starting ___ project.
6. ___ tests are in ___ /tests directory.

### Ejercicio 4: Corrige los errores de artículos

1. "I use the Python for the backend."
2. "She is developer."
3. "The React is a popular the framework."
4. "I found the bug." (primera mención)
5. "Programming the JavaScript is fun."

---

### Respuestas

**Ejercicio 1:** 1. an, 2. a, 3. an, 4. a, 5. an, 6. a, 7. an, 8. a

**Ejercicio 2:** 1. (---), 2. the, 3. (---), 4. The, 5. (---), 6. the, 7. (---), 8. The

**Ejercicio 3:**
1. a
2. (nada) / the
3. an / the
4. The / an
5. (nada) / the
6. The / the

**Ejercicio 4:**
1. "I use Python for the backend."
2. "She is a developer."
3. "React is a popular framework."
4. "I found a bug."
5. "Programming JavaScript is fun." o "Programming in JavaScript is fun."
