# Common Adjectives in Programming

## Adjetivos comunes para describir código y sistemas

Los adjetivos son palabras que describen sustantivos. En programación, los usamos constantemente para hablar sobre la calidad del código, el estado de los sistemas y las características del software.

---

## 1. Adjetivos de velocidad y rendimiento

| Adjetivo | Significado | Ejemplo |
|----------|-------------|---------|
| fast | rápido | The API is fast. |
| slow | lento | The query is too slow. |
| responsive | que responde rápido | The UI is very responsive. |
| lightweight | ligero | This is a lightweight library. |
| scalable | escalable | The architecture is scalable. |
| efficient | eficiente | This algorithm is more efficient. |

---

## 2. Adjetivos de calidad de código

| Adjetivo | Significado | Ejemplo |
|----------|-------------|---------|
| clean | limpio | The code is clean and readable. |
| dirty | sucio | This is a dirty workaround. |
| readable | legible | This function is very readable. |
| maintainable | mantenible | We need maintainable code. |
| reusable | reutilizable | Build reusable components. |
| modular | modular | The project has a modular structure. |
| robust | robusto | The error handling is robust. |
| elegant | elegante | That's an elegant solution. |

---

## 3. Adjetivos de estado del sistema

| Adjetivo | Significado | Ejemplo |
|----------|-------------|---------|
| stable | estable | The production build is stable. |
| unstable | inestable | The staging environment is unstable. |
| broken | roto | The CI pipeline is broken. |
| deprecated | obsoleto/deprecado | This method is deprecated. |
| legacy | heredado/antiguo | We have a legacy codebase. |
| outdated | desactualizado | The dependencies are outdated. |

## 5. Comparativos y superlativos

En inglés, los adjetivos cambian de forma para comparar.

### Reglas básicas

| Tipo | Comparativo | Superlativo | Regla |
|------|------------|-------------|-------|
| Cortos (1 sílaba) | + **-er** | + **-est** | fast -> faster -> fastest |
| Terminan en -e | + **-r** | + **-st** | stable -> stabler -> stablest |
| Terminan en consonante + y | **-ier** | **-iest** | dirty -> dirtier -> dirtiest |
| Largos (2+ sílabas) | **more** + adj | **most** + adj | scalable -> more scalable |
| Irregulares | forma especial | forma especial | good -> better -> best |

### Tabla de comparativos comunes en tech

| Adjetivo | Comparativo | Superlativo |
|----------|------------|-------------|
| fast | faster | fastest |
| slow | slower | slowest |
| clean | cleaner | cleanest |
| simple | simpler | simplest |
| stable | more stable | most stable |
| readable | more readable | most readable |
| efficient | more efficient | most efficient |
| scalable | more scalable | most scalable |
| good | better | best |
| bad | worse | worst |

### Ejemplos en contexto

```text
Python is slower than C, but easier to learn.
React is more popular than Svelte.
This is the fastest algorithm we've tested.
Go is simpler than Rust for this use case.
This is the worst bug we've ever had.
The refactored version is much cleaner.
This approach is more maintainable than the previous one.
```

## 6. Estructuras comparativas

### Comparativo: more/less ... than

```text
TypeScript is more verbose than JavaScript.
This query is less efficient than the previous one.
The new API is faster than the old one.
```

### Igualdad: as ... as

```text
This library is as fast as the native implementation.
The code isn't as clean as it should be.
Python is not as fast as C++.
```

### Superlativo: the most/least ... / the -est

```text
This is the most scalable architecture we've designed.
It's the simplest solution to the problem.
That was the least readable code I've ever reviewed.
```

## 7. Adjetivos que terminan en -ed vs -ing

Algunos adjetivos tienen dos formas. La diferencia es importante:

- **-ed**: describe cómo se siente alguien (el que experimenta)
- **-ing**: describe la característica de algo (lo que causa el sentimiento)

| -ed (persona) | -ing (cosa) | Ejemplo |
|---------------|-------------|---------|
| confused | confusing | The error message is **confusing**. I'm **confused**. |
| frustrated | frustrating | Debugging is **frustrating**. I feel **frustrated**. |
| interested | interesting | This project is **interesting**. I'm **interested**. |
| overwhelmed | overwhelming | The codebase is **overwhelming**. I'm **overwhelmed**. |
| deprecated | deprecating | (Contexto especial en tech) |

```text
The documentation is confusing.      (la documentación causa confusión)
I'm confused by the documentation.   (yo estoy confundido)
This bug is frustrating.             (el bug causa frustración)
I'm frustrated with this bug.        (yo estoy frustrado)
```

## 8. Adjetivos frecuentes en code reviews

```text
This code is too complex.           - Este código es demasiado complejo.
The logic is unclear.               - La lógica no es clara.
This approach is brittle.           - Este enfoque es frágil.
The naming is inconsistent.         - Los nombres son inconsistentes.
The function is too long.           - La función es demasiado larga.
This solution is clever but unreadable. - Esta solución es ingeniosa pero ilegible.
The test coverage is insufficient.  - La cobertura de tests es insuficiente.
The implementation is straightforward. - La implementación es directa/sencilla.
```
