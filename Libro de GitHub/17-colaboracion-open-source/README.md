# Capítulo 17 — Colaboración Open Source

Bienvenido al mundo del código abierto. En este capítulo aprenderás qué significa
open source, cómo contribuir a proyectos de otras personas, y las reglas no escritas
que toda la comunidad respeta. Si quieres crecer como desarrollador, contribuir a
proyectos open source es una de las mejores formas de hacerlo.

---

## ¿Qué es Open Source?

Open source (código abierto) significa que el código fuente de un proyecto está
disponible públicamente para que cualquiera pueda verlo, usarlo, modificarlo y
distribuirlo, siempre respetando la licencia del proyecto.

No es solo código gratis. Es una filosofía de colaboración donde miles de personas
trabajan juntas para construir herramientas que benefician a todos.

---

## ¿Por qué contribuir?

- **Aprendes de los mejores** — los grandes proyectos tienen código de altísima calidad
- **Mejoras tu perfil** — los reclutadores valoran contribuciones open source
- **Construyes red de contactos** — conoces desarrolladores de todo el mundo
- **Resuelves problemas reales** — no son ejercicios de tutorial, son herramientas que usan millones
- **Devuelves a la comunidad** — usas herramientas gratis todos los días, puedes aportar algo

---

## El flujo de contribución

```
1. Encuentras un proyecto que te interesa
2. Haces un fork (copia a tu cuenta)
3. Clonas tu fork a tu computadora
4. Creas una rama para tu cambio
5. Haces los cambios y commits
6. Subes tu rama a tu fork
7. Abres un Pull Request al proyecto original
8. Los mantenedores revisan tu PR
9. Si todo está bien, lo fusionan
```

---

## Licencias más comúnes

| Licencia | Carácterísticas |
|---|---|
| MIT | Muy permisiva, puedes hacer casi cualquier cosa |
| Apache 2.0 | Similar a MIT pero con protección de patentes |
| GPL v3 | Si usas el código, tu proyecto también debe ser GPL |
| BSD | Muy permisiva, similar a MIT |
| ISC | La más simple y permisiva |

**Regla de oro:** siempre respeta la licencia del proyecto al que contribuyes.

---

## Etiqueta del contribuidor

- Lee el README y CONTRIBUTING.md antes de hacer cualquier cosa
- Busca si tu problema o idea ya fue reportada en los issues
- Sé respetuoso y paciente con los mantenedores
- Escribe mensajes claros en tus commits y Pull Requests
- No tomes a mal si rechazan tu PR — aprende del feedback
- Respeta el Code of Conduct del proyecto

## Conceptos clave de este capítulo

- **Open source** — código disponible públicamente bajo una licencia abierta
- **Fork** — copia de un repositorio ajeno en tu propia cuenta de GitHub
- **Upstream** — el repositorio original del cual hiciste fork
- **Mantenedor** — persona que administra y cuida un proyecto open source
- **Contribuidor** — cualquier persona que aporta cambios a un proyecto
- **Code of Conduct** — reglas de comportamiento de una comunidad
