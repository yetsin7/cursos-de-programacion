# Capítulo 12 — Issues y Proyectos

GitHub no es solo para código — es una plataforma completa de **gestión de proyectos**. Con Issues, etiquetas, milestones y Projects puedes organizar todo el trabajo de tu equipo sin salir de GitHub.

---

## ¿Qué son los Issues?

Un **Issue** es un registro de trabajo: puede ser un bug, una nueva funcionalidad, una mejora, una pregunta o una tarea. Es la forma estándar de rastrear todo lo que hay que hacer en un proyecto.

Cada Issue tiene:
- **Título** — Descripción breve del problema o tarea
- **Cuerpo** — Descripción detallada con Markdown
- **Etiquetas** — Categorías como `bug`, `enhancement`, `documentation`
- **Asignados** — Quién se encarga de resolverlo
- **Milestone** — A qué versión o meta pertenece
- **Proyecto** — En qué tablero de gestión aparece

---

## ¿Qué son los GitHub Projects?

**GitHub Projects** es la herramienta de gestión de proyectos integrada en GitHub. Funciona como un tablero Kanban con columnas (To Do, In Progress, Done) y permite:

- Organizar Issues y PRs en un tablero visual
- Crear vistas personalizadas (tabla, tablero, línea de tiempo)
- Automatizar movimientos entre columnas
- Filtrar y agrupar por etiquetas, asignados, prioridad
- Planificar sprints y releases
